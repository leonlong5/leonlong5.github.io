Posted on Aug 4 2018 | In Front End

Introduction

Promise object is a solution for javascript asynchronous operation, provided a unified interface, It is like a proxy, act as an intermediary between asynchronous operations and callback funcitons. Pormise allows us to write asynchronous operations in synchronous way, no need to deal with callback hells.

Promise object

Let’s take a look at what a promise object looks like.

As we can see from above, the Promise construction function takes a callback function f1 as paramerter, inside the f1 is asynchronous operations. then p1 is a promise instance.

From Promise design, all asynchronous operatiosn returns a Promise instance. Promise has a then method, used to deal with next callback function.

After f1 async operation is done, the f2 method will be excuted.

##The states of Promise object

Promise has three states, used to control async opeartions.

async operation pending: pending
async operation succussed: fulfulled
async operation failed: rejected

there are only two path:

from ‘pending’ to ‘fulfilled’
from ‘pedning’ to ‘rejected’
Once state changed, it will never change again, this is also explained the name of ‘Promise’, like in real life we promise something, we either make the promise come true or failed.

##Promise construction function

there are two arguments ‘resolve’ and ‘reject’, when async operation successed, we call resolve and pass the value to it. On the other hand if failed, we call rejected and pass the error to it.
