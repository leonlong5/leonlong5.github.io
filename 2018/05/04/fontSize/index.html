Font size in css

When it comes to setting the size of font in CSS, you have lots of options: px, em, and rem, so let’s take a look at the difference and when to use it.

px

Based on computer screen, tell browser to render the letters exactly that hight of pixels in height.

  • Problem: It cannot scale when browser resize or user adjust the default font size of their web page.

em

em is based on its parent element font size, by default browser font size is 16px, so 1em = 16px. em will multiply times to the parent font size, like 2em = 32px.

  • Problem:

    1. It’s hard to calculate the font size in px, for example, if we set 1.2 em = 19.2px, decimal digits will not be take into account.
    2. Because em is based on multiple of parent font size, so nested content need to recaculate the size. For example:

    Body font-size: 62.5% is 16px*62.5%=10px

    outer span has 1.6 multiply body font size 10px = 16px

    inner span has 1.6 multiply outer span font size 16px = 25px

    rem

    Refer MDN:

    rem values are relative to the root html element, not the parent element.

    rem is used to avoid wrong font size value of nested content. It is always relative to the root html element. so for example we set html font-size=16px, if we want 12px then we need 12/16 = 0.75rem. With rem we can easily assign relative font size.
