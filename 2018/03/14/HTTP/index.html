Introduction

HTTP protocols(HyperText Transfer Protocol) is an application layer protocols that we use a lot while building a web application. It does not involving any packet transmit, it is a set of rules how client and server communicate each other.

HTTP/1.0

  1. Introduction

On May 1996, HTTP/1.0 released. First, any format of content could send over internet, not only text, but images, videos, files…etc. This laid the foundation for the thrive of the internet.

  2. Response format

  3. Content-type
    character encoding form 1.0 version, header information must be ASCII code, but the remaining could be any format, so when server response, it must let client know what format the data is, so here we have Content-Type.

Below are some common use Content-Type.

text/plain
text/html
text/css

image/png
image/svg+xml
audio/mp4
video/mp4
application/javascript
application/pdf
application/zip
application/atom+xml

These data types called MIME type.

  1. Content-encoding
    Data can be compressed before sending, Content-encoding indicate the compress method.

HTTP/1.1

Just half year later, On Jan 1997, HTTP/1.1 released. It improved a lot based on 1.0 and it’s been used until today.

  1. persistent connection

One of the problem 1.0 has is that every request needs establish TCP connection, when response send the TCP connection will be shut down. If you want to request another resource, you have to establish another connection, it is very costly when establish a TCP connection. Now the TCP connection will on until there is no activity for a while.

  1. Pipelining

1.1 introduced pipelining, which client can send multiple requests in one TCP connection. For example, if user want to request 2 resources, in 1.0 they need to send out first request, wait the response of the first then send out second request, but now the user could send them together at the same time, but the server still handle the request one by one by the order received.

  1. disadvantage
    As we can see that all the requests are handled one by one, if one request takes long time to response, it will block the requests behind it, this is so called Head-of-line blocking. In order to solve the problem, there are two ways: first is reduce the request number, second is open multiple persistent connection. Because of this, lot web optimization methods can help website loading faster appeared: merge js, css files, put icons in CSS sprite, domain sharding…etc.
